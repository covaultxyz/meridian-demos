Runbook 01 — big-white Setup for Lex Terminal

**Machine**: bacherman-caselaw (100.91.70.47 Tailscale), "big-white"

**OS**: Ubuntu 24.04, 64GB RAM, 2TB /data LV

**Caselaw Neo4j**: systemd, APOC 5.26.2 + GDS 2026.03.0, data at /data/neo4j (916GB NVMe used)

---

Pre-Flight Verification

```bash

ssh root@100.91.70.47

Verify Neo4j is running

systemctl status neo4j

Verify caselaw graph state

cypher-shell -u neo4j -p "$CASELAW_NEO4J_PASSWORD" \

"MATCH (c:Case) RETURN count(c) AS cases,

(MATCH (j:Judge) RETURN count(j)) AS judges"

Expected: ~11.08M cases, ~960K judges

```

1. System Dependencies

```bash

Python 3.11

apt-get install -y python3.11 python3.11-venv python3.11-dev build-essential

Ollama for local embeddings (GTX 1070 Ti, CUDA 12.2)

curl -fsSL https://ollama.ai/install.sh | sh

systemctl enable ollama && systemctl start ollama

ollama pull all-minilm-l6-v2 # 768d, matches caselaw HNSW index

Optional (better for legal):

ollama pull legal-bert-base-uncased

Verify GPU available

nvidia-smi

ollama run all-minilm-l6-v2 --verbose "test embedding"

```

5. Systemd Units for Lex Daemons

Lex daemons use the existing `meridian-daemon@.service` template (now has `MCP_DAEMON_NAME=%i` after GH fix commit db5b0efb1). Add Lex-specific environment overrides:

```bash

Create override directory for martin on Lex

mkdir -p /etc/systemd/system/meridian-daemon@martin.service.d/

cat > /etc/systemd/system/meridian-daemon@martin.service.d/lex.conf << 'EOF'

[Service]

EnvironmentFile=/etc/lex/env

Environment=WORKSPACE_ID=lex

Environment=MIG_NEO4J_URI=bolt://100.112.130.22:7750

Environment=CASELAW_NEO4J_URI=bolt://localhost:7687

EOF

systemctl daemon-reload

systemctl enable meridian-daemon@martin

systemctl start meridian-daemon@martin

systemctl status meridian-daemon@martin

```

Repeat for delphi, sensei, watcher, las (P0 daemons), then sandman, atlas, sterling, avery (P1).

6. Cloudflare Tunnel (KaaS API External Access)

```bash

Install cloudflared

wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb

dpkg -i cloudflared-linux-amd64.deb

Authenticate (requires operator to run in browser session first)

cloudflared tunnel login

Create tunnel for Lex KaaS API

cloudflared tunnel create lex-kaas

cloudflared tunnel route dns lex-kaas lex.speedoftrust.ai

Configure: point tunnel to KaaS FastAPI on localhost:8443

cat > /etc/cloudflared/config.yml << 'EOF'

tunnel: <tunnel-id>

credentials-file: /root/.cloudflared/<tunnel-id>.json

ingress:

• hostname: lex.speedoftrust.ai

service: http://localhost:8443

• service: http_status:404

EOF

systemctl enable cloudflared

systemctl start cloudflared

```
