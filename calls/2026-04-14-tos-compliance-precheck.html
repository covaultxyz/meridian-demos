Area 1: Automated Attribute Updates via SP-API

What Amazon Allows

The SP-API Listings Items API is explicitly designed for automated product listing management. Amazon's own documentation promotes "automated bulk submission solutions that streamline the entire workflow from catalog search and product type identification to bulk submission."

Two write operations are available:

• **`putListingsItem`** â€” Creates or fully replaces a listing. Omitted attributes are dropped.

• **`patchListingsItem`** â€” Partial update of top-level attributes only. Nested attribute patching is not supported.

Rate Limits (Listings Items API v2021-08-01)

| Operation | Per Account-App | Per Application | Burst |

|-----------|----------------|-----------------|-------|

| `getListingsItem` | 5 req/s | 100 req/s | 5 |

| `putListingsItem` | 5 req/s | 100 req/s | 5 |

| `patchListingsItem` | 5 req/s | 500 req/s | 5 |

| `deleteListingsItem` | 5 req/s | 100 req/s | 5 |

| `searchListingsItems` | 5 req/s | 100 req/s | 5 |

**Throughput estimate for Rankona:** At 5 req/s per account-app pair, updating all 43,000 SKUs individually via `patchListingsItem` takes ~2.4 hours per marketplace. For 11 marketplaces sequentially: ~26 hours. The JSON_LISTINGS_FEED bulk approach (Area 2) is the correct path.

What Amazon Prohibits

• Circumventing throttling quotas by creating multiple developer accounts (AUP Section 3.10)

• Browser automation, screen scraping, undocumented API endpoints (Agent Policy, March 2026)

• All automated actions must flow through a registered SP-API application

Compliance Requirements for the Terminal

1. **All API calls must use the registered SP-API application** â€” no direct Seller Central automation

2. **Implement token bucket rate limiting** with exponential backoff + jitter on HTTP 429

3. **Audit logging** â€” timestamps, action classifications, inputs, outputs (12-month retention minimum)

**PDR impact:** None. The architecture already routes through SP-API. Add rate limiter and audit logger to Station 1A pipeline.

Throughput Model for Rankona (43K SKUs x 11 Marketplaces)

Best Practices (from Amazon docs)

• Send one large feed rather than many small feeds

• Include only products being updated, not entire inventory

• Wait for previous feeds to complete before resubmitting

• Upload one feed of the same type no more than once every 20 minutes

2. **Track feed processing status** via `getFeed` polling â€” do not fire-and-forget

3. **Implement sequential feed submission** per type within a region

4. **Do not resubmit variation family relationships** for existing parent-child links

**PDR impact:** None. The PDR's JSON_LISTINGS_FEED approach at Station 1A-6 through 1A-8 is the correct mechanism. Feed sizing is within limits.

Area 3: AI-Generated Product Content

Amazon's Current Position

Amazon does **not** prohibit AI-generated product content. Amazon actively promotes it:

• **Enhance My Listing** (launched May 2025): Uses generative AI (Amazon Bedrock) to suggest titles, descriptions, attributes, and missing details. 900,000+ selling partners have used it with ~90% acceptance rate.

• **AI Listing Creation**: Amazon provides tools to generate listings from URLs, images, or brief descriptions.

Critical Compliance Requirement

**Human review attestation**: If you use AI-generated content (Amazon's tools or third-party), you agree that you have **reviewed the content** and that it is **accurate, complete, and compliant** with all applicable laws and Amazon policies.

This is the human-in-the-loop gate for content.

Product Listing Content Rules (Apply to All Content)

| Rule | Detail |

|------|--------|

| Title length | 200 characters max (enforced Jan 21, 2025) |

| Title characters | No !, $, ?, _, {, }, ^ |

| Title structure | Brand > Flavor/Style > Product Type > Key Attribute > Color > Size |

| Word repetition | No word repeated more than twice in title |

| Claims | Must be true, accurate, substantiated |

| Prohibited | Health/medical claims without FDA, "#1 on Amazon", unverified "Best Seller" |

| Non-compliance | 14-day correction window, then Amazon auto-corrects |

EU AI Act (7 of 11 Marketplaces)

**Deadline: August 2, 2026** â€” transparency obligations take effect.

Affected marketplaces: DE, ES, FR, IT, NL, SE, PL. (GB is post-Brexit with separate AI framework.)

Requirements for sellers ("deployers"):
