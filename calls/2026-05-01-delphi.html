Retrieval Stack

**Layer 1 — BM25 Full-Text** (primary, fast, jurisdiction-filterable)

```cypher

CALL db.index.fulltext.queryNodes('case_fulltext', $query)

YIELD node, score

WHERE ($jurisdiction IS NULL OR node.jurisdiction = $jurisdiction)

RETURN node.case_id, node.citation_str, node.title, node.court_name,

node.year, node.pagerank, score

ORDER BY score * log(1 + node.pagerank) DESC

LIMIT $limit

```

**Layer 2 — HNSW Vector** (semantic similarity, requires embeddings populated)

```cypher

MATCH (c:Case) WHERE c.embedding IS NOT NULL

WITH c, vector.similarity.cosine(c.embedding, $query_embedding) AS sim

WHERE sim > 0.75

RETURN c.case_id, c.citation_str, c.title, c.court_name, c.year, c.pagerank, sim

ORDER BY sim DESC LIMIT $limit

```

**Layer 3 — Citation Graph** (precedent chains via CITES edges)

```cypher

MATCH (c:Case {case_id: $case_id})-[:CITES*1..2]->(cited:Case)

RETURN cited.case_id, cited.citation_str, cited.title,

cited.pagerank, count(*) AS citation_depth

ORDER BY cited.pagerank DESC

```

Hybrid Scoring:

`final_score = bm25_score * 0.4 + vector_sim * 0.4 + log(1 + pagerank) * 0.2`

Dossier Assembly Pattern

Given a legal question:

1. BM25 search (top 20 candidates)

2. Vector re-rank (if embeddings available)

3. Citation chain traversal (backward 1-2 hops from top 5 results)

4. Judge profile enrichment (for cases involving key judges)

5. Contradiction scan (flag conflicting circuits on the issue)

6. Assemble: top 5-10 cases + jurisdiction summary + confidence score + limitations
