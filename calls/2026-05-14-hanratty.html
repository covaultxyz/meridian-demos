Lex-Specific Investigation Protocols

Party Research (P0 litigation use case)

Starting point: Case node from caselaw graph (INVOLVES edge gives party names).

1. Pull litigation history: `MATCH (p:Party {name: $name})-[:INVOLVES]->(c:Case) RETURN c ORDER BY c.year DESC LIMIT 20`

2. Identify patterns: plaintiff/defendant rate, courts, win/loss trends

3. Cross-reference entity in public records (EDGAR for corps, PACER for federal cases outside graph)

4. Flag: any prior adverse relationship with MERIDIAN clients (Clerk conflict check trigger)

Corporate Entity Recon

Sequence:

1. Jurisdiction of incorporation (public state records)

2. Registered agent and registered address

3. EDGAR presence (public company or prior filing history)

4. UCC filings (asset encumbrance — useful for collections and asset tracing)

5. Subsidiary/parent chain (trace via SEC ownership disclosures or state records)

6. Litigation history via caselaw graph: `MATCH (p:Party)-[:INVOLVES]->(c:Case) WHERE p.name CONTAINS $corp_name`

Expert Witness Profile

Law firms pay heavily for expert vetting. Sequence:

1. Prior testimony history in caselaw graph: search INVOLVES where party_type='expert'

2. Daubert challenges: `caselaw_search("Daubert " + expert_name + " excluded")`

3. Publication conflicts: academic affiliations vs. industry funding

4. Prior adverse testimony: cases where same expert testified for the other side

5. Rate consistency: prior fee disclosures in depositions (PACER public filings)

Opposing Counsel Profile

1. Win/loss rate in jurisdiction (DECIDED_BY + INVOLVES + caselaw outcomes)

2. Signature motions (what do they always file? summary judgment heavy? discovery tactics?)

3. Prior sanctions (Rule 11 history — searchable in full text)

4. Judge-specific history (do they have a preferred relationship with any judge on this matter?)
