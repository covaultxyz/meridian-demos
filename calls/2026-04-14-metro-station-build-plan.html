Line 1: Foundation

Station 1A — Backend Attribute Engine

**Line:** 1 (Foundation)

**What it does:** Maps category ontology once per category, then reuses it to populate all backend attributes via AI-assisted inference. Pulls category reports, reconciles against Amazon's taxonomy for gaps and errors, bulk uploads with rejection diagnostics.

Inputs required:

• Client's Amazon Seller Central access (category listing reports)

• Category taxonomy from Amazon (per marketplace)

• Client product data (sheets, specs)

• COSMO attribute requirements (15 critical backend attributes for Amazon COSMO/Rufus)

Outputs / Deliverables:

• Category ontology mapping (reusable per category)

• 100% required fields populated

• Zero rejections on bulk upload

• Rejection diagnostics report (when rejections occur during iteration)

• 60-75% time reduction vs manual process

**Dependencies:** None — this is the foundational starting point.

**Feedback connections:** Receives enrichment from Station 3 (Market Intelligence) and Station 5 (Buyer Signal Engine) as those stations come online.

MERIDIAN components needed:

• Fact extraction pipeline (brain/fact_extractor.py) for attribute inference

• Ontology mapping tools (category-specific, done once per category)

• Bulk upload validation engine

• Truth hierarchy sequencing (AI-assisted population following defined precedence)

• Flat file generation for categories that cannot be handled via API

**Estimated build time:** "Within two good days" per category batch (stated by Michael). Iterative — batches improve with each cycle. Start with top sellers, work down.

Station 1B — Listing Copy Engine

**Line:** 1 (Foundation) — concurrent with Station 1A

**What it does:** Generates SEO-first listing copy with primary keywords in title, secondary in bullets, tertiary in backend search terms. Uses embedded keyword optimization — longer phrases that contain shorter target terms to maximize keyword coverage while maintaining copy coherence.

Inputs required:

• Product sheets and brand guides

• Review and Q&A scraping (voice of customer data — NOT "Voice of the Customer" which is Amazon account health terminology)

• Keyword research (target terms, search volumes, conversion data)

• Client brand voice and positioning guidelines

• Existing listing content (for gap analysis)

Outputs / Deliverables:

• Top 50 keywords indexed per product

• >= 90% first-round approval rate

• Maximum 2 revision rounds

• SEO-optimized titles (primary keywords, embedded terms)

• Bullet points (secondary keywords + benefits)

• Backend search terms (tertiary, 250 characters, embedded keyword strategy)

**Dependencies:** None — runs concurrent with Station 1A. Both form the foundation.

**Feedback connections:** Receives enrichment from Station 4 (Competitive Position Tracking) and Station 5 (Buyer Signal Engine). Competitor intelligence feeds directly back into copy optimization.

MERIDIAN components needed:

• Listing copy generation engine

• Keyword embedding optimizer (multi-word phrase containment logic)

• Review/Q&A scraper and voice-of-customer extractor

• Copy coherence validator (ensures embedded keywords read naturally)

• Approval workflow (revision round tracking)

**Estimated build time:** 15 days (stated by Michael).

Inputs required:

• Optimized listings from Station 1B (source content)

• Backend attributes from Station 1A

• Country-specific keyword data (per marketplace)

• Local market buyer psychology insights

• Regulatory and compliance requirements per country

Outputs / Deliverables:

• Localized listings across 11 marketplaces: US, UK, DE, SE, ES, FR, IT + additional geographies

• Country-specific keyword targeting (not translated keywords)

• Native-proofed copy per market

• Compliance flags per market (especially for regulated categories like supplements)

**Dependencies:** Station 1A and Station 1B must be operational. Localization works on content produced by the foundation stations.

**Feedback connections:** Receives intelligence from Line 2 stations as they come online. Market-specific performance feeds back into localization tuning.

MERIDIAN components needed:

• Multi-market localization pipeline

Deployment approach:

1. **Do NOT start with all 43,000 SKUs.** Start with top sellers, batch process, prove workflow.

3. Group top sellers by category (e.g., if 10 of top sellers are earbuds, batch them together).

4. Each category requires ontology mapping ($2,500 one-time per category).

5. Station 1A processes backend attributes in batches — "two good days" per batch.

6. Station 1B processes listing copy concurrently — 15 days.

7. Iterate: batch 1 teaches the system, batches 2-3-4 get incrementally better.

8. Once batch system is proven and stable, throttle up volume.

Stated target (month 3):

• 40,000 SKU client live

• Category ontology mapped for product line

• Jennifer's team processes full backend in an afternoon

Why this client first:

• Immediate revenue impact — "just by fixing [the backend], we very quickly can bump up their sales and the profit"

• Profit-sharing model means Meridian earns from month 2

• Large catalog provides rich data for training the system

• Jennifer already has Seller Central access as of April 13
