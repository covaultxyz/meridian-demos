Part 0 — Pre-flight

**Before you touch the VPS.** The original install lost hours to mid-install blocking: waiting

for a Telegram token, discovering a deploy key was never added to GitHub, needing an interactive

OAuth session that couldn't run in a script. Gather everything in this section before issuing

your first SSH command.

0.1 Credentials Checklist

Have each of these available or confirmed before starting:

• [ ] VPS root SSH access confirmed (password or SSH key — you will change this)

• [ ] GitHub account with admin access to `covaultxyz/rpm-capital` (private repo)

• [ ] Telegram account — you will DM @BotFather to create the bot

• [ ] Claude.ai account capable of OAuth (for `claude auth login` — interactive, browser required)

• [ ] Tailscale account with access to the rpm-capital tailnet (separate from any MERIDIAN tailnet)

0.2 Generate Secrets Before Starting

Generate these now and store them in a local password manager. You will paste them into

`/etc/rpm-capital/env` in Part 5.

```bash

NATS auth token

openssl rand -hex 32 # → NATS_AUTH_TOKEN

Neo4j password

openssl rand -hex 16 # → NEO4J_PASSWORD

Backup encryption password

openssl rand -hex 24 # → RESTIC_PASSWORD

```

0.3 Generate the GitHub Deploy Key

> **WARNING (Gotcha #10):** The deploy key MUST be added to the GitHub repo before Part 4's

> `git clone` step. You cannot do this mid-install without a separate machine. Do it now while

> you have your local terminal and browser open.

```bash

On your LOCAL machine (not the VPS):

ssh-keygen -t ed25519 -C "rpm-capital-deploy" -f ~/.ssh/rpm_capital_deploy -N ""

cat ~/.ssh/rpm_capital_deploy.pub # copy this output

```

1. Open `https://github.com/covaultxyz/rpm-capital` → Settings → Deploy keys → Add deploy key

2. Title: `rpm-capital-master`

3. Key: paste the `rpm_capital_deploy.pub` content

4. Allow write access: NO (read-only is correct)

5. Click "Add key"

You will transfer the private key (`~/.ssh/rpm_capital_deploy`) to the VPS in Part 4.

0.4 Record the VPS IP

Note the RackNerd public IP and the Tailscale IP (you'll get the latter after Part 1).

The Tailscale IP becomes the primary access method after SSH is locked to the Tailscale interface.

Checkpoint 0

```

[ ] GitHub deploy key added to covaultxyz/rpm-capital (read-only)

[ ] NATS_AUTH_TOKEN generated and stored

[ ] NEO4J_PASSWORD generated and stored

[ ] RESTIC_PASSWORD generated and stored

[ ] Claude.ai account available for OAuth login
```

Part 1 — VPS Provisioning

Starting from a fresh Ubuntu 22.04 VPS. Sources: 01-vps-provisioning.md Phases 0–2.

1.1 Initial OS Setup

```bash

Set hostname

hostnamectl set-hostname rpm-capital-master

echo "127.0.0.1 rpm-capital-master" >> /etc/hosts

System update

apt update && apt upgrade -y

Required packages

apt install -y curl wget git vim htop iotop lsof net-tools ufw \

Default firewall posture

ufw default deny incoming

ufw default allow outgoing

```

1.2 Tailscale — Join the rpm-capital Tailnet

> **WARNING:** This is the rpm-capital tailnet, NOT any MERIDIAN or covault tailnet. Using the

> wrong tailnet will block Tailscale-restricted SSH access once you close the public SSH port.

```bash

curl -fsSL https://tailscale.com/install.sh | sh

tailscale up --advertise-tags=tag:rpm-capital

Complete the OAuth flow in your browser

tailscale ip -4 # record this IP — this becomes your primary access address

```

1.3 SSH Hardening

After confirming Tailscale is active and you can reach the node via its Tailscale IP:

```bash

Lock SSH to Tailscale interface only

ufw delete allow ssh

ufw allow in on tailscale0 to any port 22

ufw reload

Add operator SSH public key

mkdir -p /root/.ssh

echo "<your-operator-public-key>" >> /root/.ssh/authorized_keys

chmod 600 /root/.ssh/authorized_keys

```

**Test from a second terminal** via the Tailscale IP before closing your original session.

1.4 Storage Layout

> **WARNING (Gotcha #18):** `/mnt/archive` MUST be mounted before daemons start. The watcher

> daemon has this path hardcoded in its disk check cycle. If `/mnt/archive` does not exist or

> is unmounted, watcher will crash on its first disk-check run.

```bash

Identify disks

lsblk

If a secondary disk is present (e.g. /dev/sdb):

mkfs.ext4 /dev/sdb

mkdir -p /mnt/archive

echo "/dev/sdb /mnt/archive ext4 defaults 0 2" >> /etc/fstab

mount /mnt/archive

Create required directories

mkdir -p /opt/rpm-capital

mkdir -p /data/rpm-capital

mkdir -p /etc/rpm-capital

mkdir -p /mnt/archive/backups

mkdir -p /mnt/archive/dumps

mkdir -p /mnt/archive/logs

mkdir -p /var/log/rpm-capital

Permissions

chmod 750 /etc/rpm-capital

```

If no secondary disk: create `/mnt/archive` as a directory on the primary disk:

```bash

mkdir -p /mnt/archive/{backups,dumps,logs}

```

Part 2 — Infrastructure Layer

NATS, Neo4j, and Ollama. Every daemon depends on all three. Do not start any daemon until

all three are healthy and verified.

2.1 NATS Server

> **WARNING (Gotcha #11):** The `NATS_URL` for all daemons on this machine must be

> `nats://localhost:4222`, NOT the Tailscale IP or `127.0.0.1`. Using the Tailscale IP causes

> intermittent connection failures when the network interface is slow to initialize at boot.

```bash

Download latest NATS server binary

NATS_VERSION=$(curl -s https://api.github.com/repos/nats-io/nats-server/releases/latest | jq -r '.tag_name')

wget "https://github.com/nats-io/nats-server/releases/download/${NATS_VERSION}/nats-server-${NATS_VERSION}-linux-amd64.zip"

unzip "nats-server-${NATS_VERSION}-linux-amd64.zip"

mv nats-server /usr/local/bin/nats-server

chmod +x /usr/local/bin/nats-server

Create config — substitute NATS_AUTH_TOKEN from your pre-flight secrets

mkdir -p /etc/nats

cat > /etc/nats/nats.conf << 'EOF'

port: 4222

jetstream {

store_dir: /data/nats

max_memory_store: 2G

max_file_store: 10G

}

authorization {

token: "REPLACE_WITH_NATS_AUTH_TOKEN"

}

EOF

chmod 600 /etc/nats/nats.conf

mkdir -p /data/nats

systemd unit

cat > /etc/systemd/system/nats-server.service << 'EOF'

[Unit]

Description=NATS Server

After=network.target

[Service]

Type=simple

ExecStart=/usr/local/bin/nats-server -c /etc/nats/nats.conf

Restart=on-failure

RestartSec=5

[Install]

WantedBy=multi-user.target

EOF

systemctl daemon-reload

systemctl enable nats-server

systemctl start nats-server

```

Install the NATS CLI for verification:

```bash

NATS_CLI_VERSION=$(curl -s https://api.github.com/repos/nats-io/natscli/releases/latest | jq -r '.tag_name')

wget "https://github.com/nats-io/natscli/releases/download/${NATS_CLI_VERSION}/nats-${NATS_CLI_VERSION}-linux-amd64.zip"

unzip "nats-${NATS_CLI_VERSION}-linux-amd64.zip" -d nats-cli-tmp

mv nats-cli-tmp/nats /usr/local/bin/nats

chmod +x /usr/local/bin/nats

```

2.2 Neo4j

> **WARNING (Gotcha #17):** Neo4j ships with default credentials `neo4j/neo4j`. On first start

> it will prompt for a new password interactively. Set it with `neo4j-admin` BEFORE starting

> the service — failing to do so causes first-time `cypher-shell` connections to prompt

> unexpectedly and confuse automation.

> **WARNING (Gotcha #8):** Documents in this runbook set reference both port 7687 and port

> 7750. The authoritative port is **7687** (native Neo4j bolt). Port 7750 appeared in

> Docker-based deployment notes that do NOT apply here. The env var must be

> `NEO4J_URI=bolt://localhost:7687`. The number 7750 should not appear in your configuration.

> **WARNING (Gotcha #7):** APOC and GDS plugin JARs MUST be installed before Neo4j starts for

> the first time. If the JARs are absent, Neo4j starts normally but `mig_pagerank()`,

> `mig_communities()`, and graph analytics tools silently fail or error.

```bash

Add Neo4j APT repository

wget -O - https://debian.neo4j.com/neotechnology.gpg.key | gpg --dearmor -o /usr/share/keyrings/neo4j.gpg

echo "deb [signed-by=/usr/share/keyrings/neo4j.gpg] https://debian.neo4j.com stable 5" > /etc/apt/sources.list.d/neo4j.list

apt update

apt install -y neo4j

Install APOC plugin

NEO4J_VERSION=$(neo4j --version | grep -oP '\d+\.\d+\.\d+')

APOC_VERSION="5.15.0" # verify at https://github.com/neo4j-contrib/neo4j-apoc-procedures/releases

wget -P /var/lib/neo4j/plugins/ \

"https://github.com/neo4j-contrib/neo4j-apoc-procedures/releases/download/${APOC_VERSION}/apoc-${APOC_VERSION}-all.jar"

Install GDS plugin

GDS_VERSION="2.6.9" # verify at https://github.com/neo4j/graph-data-science/releases

wget -P /var/lib/neo4j/plugins/ \

"https://github.com/neo4j/graph-data-science/releases/download/${GDS_VERSION}/neo4j-graph-data-science-${GDS_VERSION}.jar"

chown -R neo4j:neo4j /var/lib/neo4j/plugins/

Configure Neo4j

cat >> /etc/neo4j/neo4j.conf << 'EOF'

Memory

server.memory.heap.initial_size=2g

server.memory.heap.max_size=4g

server.memory.pagecache.size=2g

APOC + GDS

dbms.security.procedures.unrestricted=gds.*,apoc.*

dbms.security.procedures.allowlist=gds.*,apoc.*

server.config.strict_validation.enabled=false

EOF

Set initial password BEFORE first start

neo4j-admin dbms set-initial-password "REPLACE_WITH_NEO4J_PASSWORD"

systemctl enable neo4j

systemctl start neo4j

echo "Waiting 30s for Neo4j to initialize..."

sleep 30

```

Part 4 — Repo Deployment and Python Environment

The most gotcha-dense section. Read all warnings before issuing commands.

4.1 Place the Deploy Key on the VPS

Transfer the private key you generated in pre-flight from your local machine:

```bash

On your LOCAL machine — SCP the private key to the VPS

scp ~/.ssh/rpm_capital_deploy root@<tailscale-ip>:/root/.ssh/rpm_capital_deploy

On the VPS — set permissions and configure SSH

chmod 600 /root/.ssh/rpm_capital_deploy

cat >> /root/.ssh/config << 'EOF'

Host github.com

IdentityFile /root/.ssh/rpm_capital_deploy

StrictHostKeyChecking no

EOF

chmod 600 /root/.ssh/config

Verify GitHub can see the key

ssh-keyscan github.com >> /root/.ssh/known_hosts

ssh -T git@github.com # expect: "Hi covaultxyz/rpm-capital! You've successfully authenticated..."

```

4.2 Git Clone

> **WARNING:** If this step fails with `Permission denied (publickey)`, the deploy key is not

> registered in GitHub. Return to pre-flight step 0.3. Do NOT attempt an HTTPS clone — the

> autopull script uses SSH and will break.

```bash

git clone git@github.com:covaultxyz/rpm-capital.git /opt/rpm-capital

cd /opt/rpm-capital

git log --oneline -3 # verify commits

```

4.3 Python Version Check

> **WARNING (Gotcha #15):** The daemon code uses Python 3.11+ type annotations. Ubuntu 22.04

> ships Python 3.10 by default. Importing any daemon on 3.10 fails with `TypeError` or

> `SyntaxError`. Check your version and upgrade if needed before creating the virtualenv.

```bash

python3 --version # must be 3.11.x or higher

If 3.10 or earlier:

add-apt-repository ppa:deadsnakes/ppa

apt update

apt install -y python3.11 python3.11-venv python3.11-dev

update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.11 1

python3 --version # re-verify: 3.11.x

```

4.4 Create the Virtualenv

> **WARNING:** The virtualenv MUST be at `/opt/rpm-capital/venv/`. This exact path is hardcoded

> in `scripts/daemon-wrapper.sh` line 11 (`/opt/rpm-capital/venv/bin/python3`) and in

> `scripts/mcp-server.sh`. Using `.venv/`, `env/`, or any other name will silently break every

> daemon start with no helpful error message.

```bash

cd /opt/rpm-capital

python3.11 -m venv venv # or: python3 -m venv venv (if 3.11 is default)

source venv/bin/activate

pip install --upgrade pip

```

4.5 Install Dependencies

> **WARNING (Gotcha #16):** `structlog` is a required runtime dependency that is NOT in

> `requirements.txt` but IS in `pyproject.toml`. Using `pip install -r requirements.txt` will

> result in `ModuleNotFoundError: No module named 'structlog'` when daemons start. Always use

> `pip install -e .` to install from pyproject.toml, which includes all dependencies.

```bash

pip install -e .

Verify critical packages are present

python -c "import structlog, nats, neo4j, httpx, mcp; print('All core imports OK')"

```

4.6 Check the daemons Package Init File

> **WARNING (Gotcha #2/#5):** The `daemons/` directory is a Python package and MUST have an

> `__init__.py` file. Without it, `python -m daemons.las` fails with `ModuleNotFoundError`.

> The file exists in the repo but can be accidentally absent after a partial clone or a failed

> merge. Check it explicitly.

```bash

ls /opt/rpm-capital/daemons/__init__.py # must exist

```

If missing: `touch /opt/rpm-capital/daemons/__init__.py`

4.7 Smoke Test Imports

```bash

cd /opt/rpm-capital

source venv/bin/activate

python -m pytest tests/ -x -q

```

All tests must pass. This validates that all daemon classes are importable, `brain.validation_core`

is available, and the MCP server module loads without crashing.

Part 5 — Environment Configuration

All 14 environment variables in one file. Every daemon, systemd unit, and script sources this

file. It is the single source of truth for runtime configuration.

5.1 Create /etc/rpm-capital/env

> **WARNING (NATS_URL):** Must be `nats://localhost:4222`. Not the Tailscale IP. Not

> `127.0.0.1`. The `localhost` form resolves correctly through all startup orderings.

> some older documentation was a Docker container port-mapping artifact. It does not apply to

> this installation. Do not use 7750.

> **WARNING (PYTHONPATH):** Must include `/opt/rpm-capital` so that `import daemons.las` and

> `import brain.validation_core` resolve correctly regardless of how the daemon is invoked.

Part 6 — Systemd Unit Installation

6.1 Service Naming Clarification

> **WARNING (Gotcha #1/#3):** The service unit files are named `rpm-las.service`,

> `rpm-sensei.service`, etc. — individual named units, one per daemon. Some older docs

> reference a `rpm-daemon@.service` template pattern. That template is NOT what is deployed

> and NOT what `scripts/deploy.sh` and `scripts/restart-all.sh` use. Those scripts call

> `systemctl restart rpm-${d}.service`, which requires the individual named units.

> Never use the template form `rpm-daemon@las.service`.

6.2 Create Daemon Service Units

Create one unit file per daemon. Replace `<daemon>` with the daemon name each time:

```bash

for DAEMON in las sensei delphi watcher sandman atlas avery sterling martin; do

cat > /etc/systemd/system/rpm-${DAEMON}.service << UNITEOF

[Unit]

Description=RPM Capital ${DAEMON} daemon

After=network.target nats-server.service neo4j.service

Wants=nats-server.service neo4j.service

[Service]

Type=simple

EnvironmentFile=/etc/rpm-capital/env

WorkingDirectory=/opt/rpm-capital

ExecStart=/opt/rpm-capital/scripts/daemon-wrapper.sh ${DAEMON}

Restart=on-failure

RestartSec=30

StandardOutput=journal

StandardError=journal

SyslogIdentifier=rpm-${DAEMON}

[Install]

WantedBy=multi-user.target

UNITEOF

done

```

6.3 Create the Telegram Bot Unit

The Telegram bot is separate because it does not use `daemon-wrapper.sh`:

```bash

cat > /etc/systemd/system/rpm-telegram.service << 'EOF'

[Unit]

Description=RPM Capital Telegram bot

After=network.target nats-server.service

Wants=nats-server.service

[Service]

Type=simple

EnvironmentFile=/etc/rpm-capital/env

WorkingDirectory=/opt/rpm-capital

ExecStart=/opt/rpm-capital/venv/bin/python3 -m daemons.telegram_bot

Restart=on-failure

RestartSec=30

StandardOutput=journal

StandardError=journal

SyslogIdentifier=rpm-telegram

[Install]

WantedBy=multi-user.target

EOF

```

6.4 Enable Units (Do Not Start Yet)

```bash
systemctl daemon-reload

for DAEMON in las sensei delphi watcher sandman atlas avery sterling martin; do
```

Part 7 — Neo4j Indexes and JetStream Streams

Create all database schema and message bus resources before starting any daemon.

7.1 Neo4j Full-Text Indexes (BM25)

```bash

source /etc/rpm-capital/env

cypher-shell -u neo4j -p "$NEO4J_PASSWORD" << 'CYPHER'
```

Part 14 — Autopull and Deployment Automation

14.1 Create the Autopull Script

> **WARNING (Gotcha #12):** A naive `git pull` will NOT restart daemons when new code arrives.

> The autopull script must detect whether the HEAD commit changed and restart services when it

> did. A pull without restart means code updates never take effect until manual intervention.

```bash

cat > /usr/local/bin/rpm-autopull.sh << 'PULLEOF'

#!/usr/bin/env bash

set -euo pipefail

cd /opt/rpm-capital

PRE_PULL=$(git rev-parse HEAD)

GIT_SSH_COMMAND="ssh -i /root/.ssh/rpm_capital_deploy" \

git pull --rebase origin main 2>&1 | logger -t rpm-autopull

POST_PULL=$(git rev-parse HEAD)

if [[ "$PRE_PULL" != "$POST_PULL" ]]; then

logger -t rpm-autopull "New commit ${POST_PULL} — restarting daemons"

source /opt/rpm-capital/venv/bin/activate

pip install -e . --quiet

systemctl daemon-reload

bash /opt/rpm-capital/scripts/deploy.sh

fi

PULLEOF

chmod +x /usr/local/bin/rpm-autopull.sh

```

14.2 Create the Cron Entry

```bash

cat > /etc/cron.d/rpm-autopull << 'EOF'

*/15 * * * * root GIT_SSH_COMMAND="ssh -i /root/.ssh/rpm_capital_deploy" /usr/local/bin/rpm-autopull.sh 2>&1 | logger -t rpm-autopull

EOF

```

17.2 Common Failure Modes and Fixes

| Error / Symptom | Cause | Fix |

|---|---|---|

| `nats: connection refused` | NATS not running | `systemctl restart nats-server && sleep 5 && systemctl restart rpm-las` |

| `ServiceUnavailable: Connection refused` (Neo4j) | Neo4j not ready | `systemctl restart neo4j && sleep 30 && systemctl restart rpm-las rpm-delphi` |

| `BLOCKED: <daemon> not in daemons.allow` | Missing from allowlist | Add daemon name to `/etc/rpm-capital/daemons.allow`, restart |

| `ModuleNotFoundError: No module named 'daemons'` | PYTHONPATH not set | Verify `PYTHONPATH=/opt/rpm-capital` in `/etc/rpm-capital/env` |

| `ModuleNotFoundError: No module named 'structlog'` | Wrong install method | `source venv/bin/activate && pip install -e .` |

| `TypeError` on daemon import | Python 3.10 not 3.11 | Install Python 3.11 via deadsnakes PPA, recreate venv |

| Bot sends no response | Claude CLI not authenticated | SSH to VPS, `claude auth login` |

| Bot sends "claude exited 1" | Claude OAuth token expired | SSH to VPS, `claude auth login` |

| Telegram alert storm | Cooldown bypass | Check all `_send_telegram` calls use `_is_cooled_down()` |

| `backup marker missing` alert from Sandman | Backup script never ran | `bash /opt/rpm-capital/scripts/backup.sh` |

| JetStream publishes silently dropping | Streams not created | Run all `nats stream add` commands from Part 7 |

| Daemon claims `no responders available for request` | Target daemon not up yet | Check if target daemon is active, start it, wait 10s |

17.6 Restore from Backup (Full Node Recovery)

RTO target: < 2 hours from node loss.

```bash

1. Provision a new VPS and complete Parts 0–6 of this runbook

2. Restore /etc/rpm-capital/env from backup (contains all secrets)

restic restore latest --target / --include /etc/rpm-capital --repo /mnt/archive/backups/rpm-capital

3. Restore Neo4j dump

restic restore latest --target /mnt/archive/dumps --include /mnt/archive/dumps --repo /mnt/archive/backups/rpm-capital

systemctl stop neo4j

neo4j-admin database load neo4j --from-path=/mnt/archive/dumps/ --overwrite-destination=true

systemctl start neo4j && sleep 30

4. Restore /opt/rpm-capital (config, scripts — not venv)

restic restore latest --target / --include /opt/rpm-capital --exclude /opt/rpm-capital/venv

5. Recreate venv (venv is not backed up — must be rebuilt)

cd /opt/rpm-capital && python3.11 -m venv venv && source venv/bin/activate && pip install -e .

6. Start all daemons (Part 9 procedure)

```

18.5 Common Error Lookup

| Error / Symptom | Cause | Fix |

|---|---|---|

| `nats: no servers available` | NATS not running | `systemctl restart nats-server` |

| `nats: connection refused` | NATS down or wrong port | Verify NATS_URL is `nats://localhost:4222` |

| `nats: no responders available for request` | Target daemon not subscribing | Start target daemon, wait 10s |

| `ServiceUnavailable: Connection refused` (Neo4j) | Neo4j not ready | `systemctl restart neo4j && sleep 30` |

| `BLOCKED: <name> not in daemons.allow` | Daemon not in allowlist | Add name to `/etc/rpm-capital/daemons.allow` |

| `ModuleNotFoundError: No module named 'daemons'` | Missing PYTHONPATH | Verify `PYTHONPATH=/opt/rpm-capital` in env |

| `ModuleNotFoundError: No module named 'structlog'` | Installed from requirements.txt | `pip install -e .` in venv |

| `TypeError: X | Y` on import | Python 3.10 union syntax | Install Python 3.11 from deadsnakes |

| `Permission denied (publickey)` on git | Deploy key not in GitHub | Add key at covaultxyz/rpm-capital > Settings > Deploy keys |

| Bot silent / no response | Claude CLI not authed | SSH to VPS, `claude auth login` |

| Bot says "claude exited 1: not authenticated" | OAuth token expired | SSH to VPS, `claude auth login` |

| Sandman fires backup-overdue alert | Backup never ran | `bash /opt/rpm-capital/scripts/backup.sh` |

| JetStream publishes silently lost | Streams not created | Run all `nats stream add` commands from Part 7 |

| Watcher crashes on startup | `/mnt/archive` not mounted | Mount archive disk, `systemctl restart rpm-watcher` |

| Daemons show "BLOCKED" then inactive | Missing `daemons.allow` file | Create `/etc/rpm-capital/daemons.allow` with 9 names |

| All daemons fail within 5s of start | PYTHONPATH not in env | Add `PYTHONPATH=/opt/rpm-capital` to `/etc/rpm-capital/env` |
