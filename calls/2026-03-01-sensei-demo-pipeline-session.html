Pipeline Architecture

```

Cursor agent finishes feature

│

▼

Writes demos/playwright_<feature>.py (copy of playwright_template.py)

│

▼

python workers/demo_recorder.py \

--script demos/playwright_<feature>.py \

--pr <N> \

--title "<feature name>"

│

├─ 1. Snapshot demos/raw/ (capture .webm UUID diff)

├─ 2. Run Playwright with RECORD_VIDEO=1 → UUID-named .webm

├─ 3. ffmpeg: .webm → 1280x720 h264 .mp4 (scale + pad)

├─ 4. ffmpeg: drawtext title overlay (falls back if libfreetype missing)

├─ 5. gh release create demo-videos (idempotent, || true)

├─ 6. gh release upload --clobber → CDN URL

└─ 7. gh pr comment <N> --body "🎬 Demo: <CDN_URL>"

```

Videos never land in the git tree — they live as GitHub release assets on the `demo-videos` tag.

Cursor Agent Queue System

Problem Found

`~/cursor_queue.json` was write-only — nothing ever marked tasks `in_progress` or `done`. Stale `pending` entry from Feb 28 (#61, already deployed) was sitting unresolved.

Fix

`scripts/cursor_claim.py` — queue manager with subcommands:

```bash

python scripts/cursor_claim.py list # see all tasks by status

python scripts/cursor_claim.py claim <task_id_prefix> # mark in_progress (open session)

python scripts/cursor_claim.py done <task_id_prefix> # mark done (PR merged)

python scripts/cursor_claim.py abandon <task_id_prefix> # return to pending (aborted)

python scripts/cursor_claim.py enqueue --title "..." --prompt-file "..." --issue N

```

`enqueue` publishes to NATS JetStream → `cursor_dispatcher` picks it up → writes to queue file + Telegram notification + GitHub issue comment.

Parallel Session Protocol

• `cursor_claim.py list` before starting — check nothing is already `in_progress` in the same file zone

• Max 2 parallel sessions with non-overlapping file scopes

• `claim` when opening Cursor, `done` when PR merges
