Fix Boardroom

INTERNAL 2026-04-07 alex

Summary

The transcript details numerous functional and user experience gaps in the current AI agent system across prospect, sales, technical, product, and competitive perspectives, concluding with a prioritized list of five key improvements.

Decisions Made

  • Prioritize implementing a warm greeting for returning visitors that references their name and last topic.
  • Prioritize surfacing personal links via a UI button or agent utterance for conversation sharing.
  • Prioritize improving the Live A2A agent switch UX with a transfer line, transition sound, and UI updates.
  • Prioritize developing a post-call summary page for each session, including topics, action items, and personal links.
  • Prioritize building an observer UI page for live demos to show real-time transcript and conversation details.

Key Discoveries

  • Live A2A reconnects introduce a 2-3 second silent gap and voice change, making agent handoffs feel like dropped calls.
  • Users receive no feedback when documents are uploaded, leaving them unsure if the process was successful.
  • Personalized conversation links are generated automatically but are invisible to visitors and agents.
  • Returning visitors are met with a generic greeting, despite the system successfully loading their prior conversation context.
  • Refreshing the browser mid-conversation results in the complete loss of the session, transcript, and agent state.
  • Document uploads during Live A2A sessions silently fail because they incorrectly trigger a Classic mode endpoint.
  • The sales team lacks an observer dashboard to monitor prospect interactions in real-time during live demos.
  • Key technical components like the Cloudflare tunnel and voice gateway are single points of failure, lacking resilience mechanisms.
  • Session data and contact extraction are vulnerable to loss upon service restarts due to in-memory storage or missing retry logic.
  • The system's unique competitive advantages, such as multi-agent handoff and persistent memory, are not visible or effectively demonstrated to users or buyers.

LENS — examining what we built today against what a

prospect actually experiences:

The voice gap is still audible. Classic mode voices are

fixed (distinct Edge-TTS voices per agent), but the Live

A2A reconnect creates a 2-3 second dead silence while the

Gemini session tears down and rebuilds. A prospect

hears: Victoria talking → silence → different voice

starts. No transition audio, no "I'm transferring you"

spoken before the disconnect. The handoff feels like a

dropped call, not a warm transfer.

Upload has no feedback loop. Documents upload and chunks

get stored, but the agent never acknowledges the upload.

In Classic mode the document context is silently injected

into the system prompt — the agent doesn't say "I've

reviewed your document" because it doesn't know the

document just arrived. The user uploads a PDF and wonders

if it worked.

Personal links are invisible to the visitor. We generate

tokens automatically after sessions, but nobody ever sees

them. There's no UI element that says "Bookmark this

link to continue later" or "Text me this link." The agent

can't share it because there's no tool for that. The

link exists in Neo4j but never reaches the human.

The greeting is cold for returning visitors. Context

loads (we verified — 1,645 chars for Angelo), but the

agent's greeting is the same scripted opener every time.

Victoria says "Good afternoon, this call may be

recorded..." to Angelo Robles on his 5th visit. She

should say "Welcome back, Angelo" and reference the last

conversation.

No conversation persistence on page refresh. If you

refresh the browser mid-conversation, the session dies.

The transcript, the agent state, the upload context — all

gone. The visitor starts over. This is especially

painful on mobile where Safari kills background tabs.

Live A2A has no upload path. The upload button triggers

the Classic mode endpoint which requires a Classic

session_id. In Live mode there's no session_id in the

same format. Uploading during a Live session silently

fails.

PRISM — decomposing across five perspectives:

1. Prospect Experience (Angelo in the room)

What works: He opens his personal link, Victoria knows

his name and stage, she references wealth tax concerns

from last time. Four distinct voices across departments.

Upload a family office document, Classic mode agents

reference it.

What's missing:

• No mobile-optimized layout. The sidebar roster is

hidden on mobile but the mic button positioning is

awkward — the dock sits at the bottom with stats nobody

cares about (STT latency, confidence %), eating screen

real estate

• No visual indicator of which agent is currently

speaking in Live A2A. The Classic mode shows the active

agent badge, but Live A2A just shows "Live with Victoria"

and never updates the name when agents switch

• The conversation transcript in Live A2A accumulates but

doesn't show agent names — just "user" and "agent."

Can't tell which persona said what in review

• No way to end the conversation gracefully. There's no

"End Call" button. The visitor just... closes the tab. No

summary, no follow-up email, no "here's what we

discussed"

2. Sales Team Experience (Alexander demoing to a

prospect)

What works: Six boardrooms all load, all teams

configured, practice mode exists for training.

What's missing:

• No observer dashboard for live demos. The /observe SSE

endpoint exists but there's no UI for it. Alexander can't

watch what the prospect is saying in real time on a

separate screen during a demo

• No way to intervene. If Victoria says something wrong

during a live demo, there's no "whisper" mode to correct

the agent mid-conversation

• No post-call summary accessible via UI. The data is in

SQLite (Call + Transcript) but there's no page to review

what happened. You'd have to SSH into the VPS and query

the database

• The boardroom CSV is a local file. Should be a

/boardroom/directory page that lists all public

boardrooms with live links

3. Technical Reliability

What's fragile:

• The Cloudflare tunnel (meridian-sot) is a single point

of failure. If cloudflared crashes, all boardroom traffic

dies. No health check or auto-restart beyond systemd

• The voice gateway runs one uvicorn worker (no --workers

flag). A single long-running Live A2A session with

Gemini could block other requests

• The Gemini API key is a single credential. Rate limits

on Live API could block all concurrent Live sessions

• Edge-TTS is free but unreliable — no SLA, occasional

503s. The fallback chain (primary TTS → Edge-TTS

fallback) exists but hasn't been tested under load

• SQLite on VPS is a single file. Concurrent writes from

multiple boardroom sessions could hit write locks. No WAL

mode configured

• The _sessions dict is in-memory. Service restart = all

active sessions lost. No session recovery

4. Product Completeness

Half-built features:

• SLT validation pipeline — Upload works, but the 4-stage

TransductionValidator + OmissionDetector only triggers

via explicit /slt-validate endpoint. No UI button for it

• Call grading — The "Grade My Call" button in practice

mode calls showPracticeGrade() which sends the transcript

to the LLM for scoring. But it uses a hardcoded grading

rubric that's generic, not customized per boardroom

• Contact extraction runs async after sessions but

there's no guarantee it completes. If the service

restarts during extraction, the data is lost. No retry

mechanism

• Interaction nodes — The fetch_person_context function

looks for PARTICIPATED_IN → Interaction but the boardroom

doesn't consistently create Interaction nodes. Angelo

shows "0 prior conversations" despite having 37 memories.

The memory exists but the interaction summary doesn't

5. Competitive Differentiation

What a buyer would compare against (Bland.ai, Vapi,

Retell):

• Those platforms have phone number integration (Twilio).

Our boardroom is browser-only. The "Twilio phone demo

for Angelo" from the checkpoint is still undone

• Those platforms have analytics dashboards. We have raw

data but no visualization

• Those platforms have multi-language support. Our agents

are English-only (hardcoded recognition.lang = 'en-US')

• Those platforms have webhook integrations. Our

boardroom is self-contained — no CRM push, no Zapier, no

email triggers after calls

• What we have that they don't: Multi-agent handoff with

distinct voices, persistent cross-session memory,

document upload + RAG, personal identity links, the

entire MIG knowledge graph backing every agent. This is

the moat — but it's invisible without a dashboard to show

it

Top 5 highest-impact gaps to close next:

1. Warm greeting for returning visitors — If person

context loaded, modify the greeting prompt to reference

their name and last topic. ~20 lines of code in the

greeting generation path. Biggest demo wow-factor per

effort.

2. Personal link surfacing — Add a "Share this

conversation" button to the UI that shows the

personalized URL. Or have the agent say it at call close.

The infrastructure is built, it just needs a front door.

3. Live A2A agent switch UX — Before disconnecting the

old Gemini session, have the current agent say a transfer

line ("Let me connect you with Angelo for strategy").

Play a brief transition sound. Update the UI with the new

agent name. Right now the switch is silent and

invisible.

4. Post-call summary page — When a session ends, generate

a summary and show it in the UI (or redirect to a

/call/<id>/summary page). Include: topics discussed,

action items, personal link for next visit. This turns

every boardroom call into a lead artifact.

5. Observer UI for live demos — Build a simple page at

/observe/<session_id> that shows the real-time

transcript, active agent, buying signals, and

conversation phase. Alexander opens this on his laptop

while the prospect talks on theirs.
