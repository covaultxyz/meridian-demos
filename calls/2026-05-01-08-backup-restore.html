Lex Terminal — Backup & Restore Runbook

**Backup destination**: gateway8 (100.73.1.114 Tailscale), `/srv/archive-2/lex/`

**Method**: borg over Tailscale SSH

**Schedule**: Daily 03:00 UTC

**Scope**: Caselaw Neo4j dump + Lex config/venv + secrets

**CRITICAL**: Caselaw Neo4j is Community Edition — requires OFFLINE dump.

Stop Delphi and Martin BEFORE dumping. Restart after backup completes.

**CRITICAL**: Run the backup manually ONCE before enabling Sandman monitoring.

Sandman checks for the last_success marker. Without it, it fires hourly alerts

("backup never run"). Create the marker by running backup.sh manually first.

---

Why Borg (Not Restic)

gateway8 uses borg for all mesh backups. `/srv/archive-2` is already provisioned.

Introducing restic would create backup architecture inconsistency. Borg it is.

Phase 1 — SSH Key Setup (big-white → gateway8)

```bash

On big-white — generate a dedicated backup key

ssh-keygen -t ed25519 -C "lex-backup@big-white" -f /root/.ssh/lex_backup_key -N ""

Authorize on gateway8

ssh-copy-id -i /root/.ssh/lex_backup_key.pub root@100.73.1.114

Test the connection

ssh -i /root/.ssh/lex_backup_key root@100.73.1.114 'hostname && df -h /srv/archive-2'

Expected: gateway8 hostname + archive disk stats

```

Phase 2 — Borg Repository Initialization

```bash

On big-white

export BORG_PASSPHRASE="$(cat /etc/lex/backup-passphrase)"

(Generate and store backup passphrase in /etc/lex/backup-passphrase, mode 600)

echo "$(openssl rand -base64 32)" > /etc/lex/backup-passphrase && chmod 600 /etc/lex/backup-passphrase

Initialize the borg repo on gateway8

borg init \

--encryption=repokey \

root@100.73.1.114:/srv/archive-2/lex/

List repos to confirm

ssh root@100.73.1.114 'ls /srv/archive-2/'

Expected: lex/ visible in listing

```

Store the BORG_PASSPHRASE in `/etc/lex/env`:

Phase 3 — Backup Script

Create `/opt/lex/scripts/backup.sh`:

```bash

#!/usr/bin/env bash

set -euo pipefail

source /etc/lex/env

LOG_PREFIX="[lex-backup $(date -Iseconds)]"

DUMP_DIR=/tmp/lex-neo4j-dump

echo "$LOG_PREFIX Starting Lex backup"

=== Step 1: Stop Delphi and Martin (Neo4j community requires offline dump) ===

echo "$LOG_PREFIX Stopping daemons that hold Neo4j connections"

systemctl stop meridian-daemon@delphi meridian-daemon@martin || true

sleep 5

=== Step 2: Stop Neo4j and dump ===

echo "$LOG_PREFIX Dumping caselaw Neo4j (offline dump)"

systemctl stop neo4j

sleep 5

rm -rf $DUMP_DIR

mkdir -p $DUMP_DIR

neo4j-admin database dump neo4j --to-path=$DUMP_DIR/

=== Step 3: Restart Neo4j immediately after dump ===

echo "$LOG_PREFIX Restarting Neo4j"

systemctl start neo4j

sleep 20 # Wait for full startup

systemctl start meridian-daemon@las

sleep 3

systemctl start meridian-daemon@delphi meridian-daemon@martin

=== Step 4: Borg archive ===

echo "$LOG_PREFIX Creating borg archive"

export BORG_RSH="ssh -i $BORG_SSH_KEY"

borg create \

--compression lz4 \

--stats \

"$BORG_REPO::lex-$(date +%Y%m%d-%H%M)" \

$DUMP_DIR \

/opt/lex/ \

/etc/lex/ \

--exclude '/opt/lex/.venv/lib/python*/site-packages/pip*' \

--exclude '*.pyc' \

--exclude '__pycache__'

=== Step 5: Prune old backups ===

borg prune \

"$BORG_REPO" \

--keep-daily 7 \

--keep-weekly 4 \

--keep-monthly 3 \

--stats

=== Step 6: Verify repo integrity ===

borg check "$BORG_REPO"

=== Step 7: Update last_success marker ===

MARKER=/opt/lex/backups/last_success

mkdir -p /opt/lex/backups

date -Iseconds > $MARKER

echo "$LOG_PREFIX Backup complete"

```

```bash

chmod +x /opt/lex/scripts/backup.sh

```

Phase 5 — Systemd Timer

```bash

cat > /etc/systemd/system/lex-backup.service << 'EOF'

[Unit]

Description=Lex Terminal Daily Backup

After=network.target

[Service]

Type=oneshot

EnvironmentFile=/etc/lex/env

ExecStart=/opt/lex/scripts/backup.sh

StandardOutput=journal

StandardError=journal

SyslogIdentifier=lex-backup

TimeoutStartSec=7200

EOF

cat > /etc/systemd/system/lex-backup.timer << 'EOF'

[Unit]

Description=Lex Terminal Daily Backup Timer

[Timer]

OnCalendar=*-*-* 03:00:00 UTC

Persistent=true

[Install]

WantedBy=timers.target

EOF

systemctl daemon-reload

systemctl enable lex-backup.timer

systemctl start lex-backup.timer

Verify timer is active

systemctl status lex-backup.timer

systemctl list-timers | grep lex-backup

```

Restore Procedure (Disaster Recovery)

```bash

source /etc/lex/env

List available snapshots

export BORG_RSH="ssh -i $BORG_SSH_KEY"

borg list "$BORG_REPO"

Extract latest snapshot to temp dir

borg extract \

"$BORG_REPO::$(borg list --last 1 --short "$BORG_REPO")" \

--target /tmp/lex-restore/

Stop all daemons

for d in avery sterling atlas sandman watcher sensei martin delphi las; do

systemctl stop meridian-daemon@$d

done

systemctl stop neo4j

Restore Neo4j dump

neo4j-admin database load neo4j \

--from-path=/tmp/lex-restore/tmp/lex-neo4j-dump/ \

--overwrite-destination

Restore config (don't overwrite /etc/lex/env — secrets are unchanged)

rsync -av --exclude='env' /tmp/lex-restore/etc/lex/ /etc/lex/

Start Neo4j + daemons

systemctl start neo4j

sleep 20

systemctl start meridian-daemon@las

sleep 3

for d in delphi martin sensei watcher; do

systemctl start meridian-daemon@$d

done

sleep 5

for d in sandman atlas sterling avery; do

systemctl start meridian-daemon@$d

done

Verify

cypher-shell -u neo4j -p "$CASELAW_NEO4J_PASSWORD" "MATCH (c:Case) RETURN count(c);"

Expected: ~11080000

```

```bash

export BORG_RSH="ssh -i $BORG_SSH_KEY" && \

borg check $BORG_REPO && \
```
