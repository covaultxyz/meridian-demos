Rankona Mazon — Production Schedule

**Effective from:** Day 1 of terminal deployment

**Review cadence:** Monthly (first Monday of each month)

This document defines the daily, weekly, and monthly operational rhythm for the Rankona Mazon MERIDIAN terminal. All times are CET (Central European Time).

---

Daily Optimization Cycle (Weekdays)

| Time (CET) | Station | What happens | Who/What |

|------------|---------|-------------|---------|

| 06:00 | Auto | Forge pulls fresh SP-API data: inventory snapshot, session counts, unit sales | Forge daemon |

| 06:30 | 1A CARTOGRAPHER | New ASINs cataloged against category attribute requirements; attribute gaps scored and queued | Forge daemon |

| 07:00 | 3 SCOUT Intel | Competitor price changes and BSR shifts from prior 24h flagged in Neo4j | Atlas daemon |

| 07:30 | 4 SCOUT Comp. | Competitive position deltas above threshold dispatched to @RankonaBot as alerts | Delphi → @RankonaBot |

| 08:00 | 7 NAVIGATOR Ads | Yesterday's advertising spend vs. budget pacing report posted to Jennifer's Telegram chat | Delphi → @RankonaBot |

| 09:00 | Work window | Jennifer logs in; reviews alerts and approves any queued listing changes | Jennifer |

| 09:00–17:00 | All | Jennifer actions: `/queue` (pending copy), `/approve [ASIN]`, `/reject [ASIN]`, `/alerts` | Jennifer |

| 17:00 | 1B HERALD | Approved copy revisions compiled into upload batch; Forge prepares bulk file | Forge daemon |

| 17:30 | 1B HERALD | Upload batch submitted to Seller Central via SP-API | Forge daemon |

| 22:00 | Auto | Pulse report sent to Carl: sessions, units ordered, ACoS 7-day trend, listing queue depth | Delphi → @RankonaBot |

Alert thresholds (configurable):

• BSR shift >15% in 48h → immediate alert

• Competitor price undercut >10% → immediate alert

• Listing suppression detected → immediate alert (P0)

• Upload batch rejection rate >20% → alert to Jennifer + Carl

Monday Sprint Planning Format (30 min)

1. Review previous week's upload results (approved / rejected / pending Amazon processing)

2. Select 5 priority ASINs for the coming week (top by revenue, or by attribute gap score)

3. Flag any upcoming promotions or launch windows that need listing prep

4. Confirm any advertising budget changes

5. Carl approves the week's focus — Alexander configures queue priorities in MERIDIAN
