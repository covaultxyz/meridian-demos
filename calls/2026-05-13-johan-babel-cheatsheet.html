@RankonaBot — Johan's Quick Reference (Localization Role)

**Role:** `localization` — 12 commands, Station 2 (BABEL) only

---

What Station 2 Does

MERIDIAN generates Amazon listing copy in English first (Station 1B), then BABEL translates to each target market. You review translations before they go live. Your job is to confirm each translation is natural, marketplace-appropriate, and accurate for the product category.

**Active target markets for localization:** DE, ES, FR, IT, NL, SE, PL (and optionally US, BR, CA)

---

Your 12 Commands

Core Workflow

| Command | Example | What it does |

|---------|---------|-------------|

| `/babel queue` | `/babel queue` | All translations pending your review — your main starting point |

| `/babel translate [ASIN] [market]` | `/babel translate B09XYZ1234 SE` | Request a new translation for this ASIN + market |

| `/babel approve [ASIN] [market]` | `/babel approve B09XYZ1234 SE` | Approve translation — triggers upload to that marketplace |

| `/babel reject [ASIN] [market] [reason]` | `/babel reject B09XYZ1234 SE sounds robotic` | Reject with feedback — returns for revision |

| `/babel status [ASIN]` | `/babel status B09XYZ1234` | Translation status across all markets for this ASIN |

Review and Editing

| Command | Example | What it does |

| `/babel compare [ASIN] [market]` | `/babel compare B09XYZ1234 SE` | Side-by-side: English original vs. proposed translation |

| `/babel edit [ASIN] [market]` | `/babel edit B09XYZ1234 SE` | Enter edit mode — submit corrected copy line by line |

Queue Management

| Command | What it does |

|---------|-------------|

| `/babel proofing` | Show ASINs awaiting native proofer review (if assigned) |

| `/babel priority [ASIN]` | Flag ASIN as high priority in the localization queue |

| `/babel markets` | Show active target markets and localization completion % |

| `/babel history [ASIN]` | Full translation history for an ASIN across all markets |

---

Typical Session (15–20 minutes)

1. `/babel queue` — see what's pending

2. For each ASIN in queue: `/babel view [ASIN] [market]` to read the translation

3. `/babel approve [ASIN] [market]` if it reads naturally, or `/babel reject [ASIN] [market] [reason]` with specific feedback

4. `/babel status [ASIN]` to confirm the ASIN is progressing across markets

---

Rejection Feedback Guide

The rejection reason trains the system. Be specific:

| What's wrong | Good rejection reason |

|-------------|----------------------|

| Translation is too literal | "too literal, reads like machine translation" |

| Wrong register (too formal for the market) | "too formal for Swedish market, should be casual" |

| Incorrect product terminology | "wrong term — Swedish for saddle pad is 'schabrak', not 'sadelpude'" |

| Title too long for the market | "title too long, DE market limit is 80 chars" |

| Keywords don't match how locals search | "Swedish customers search for 'ridbyxor', not 'ridningsbukse'" |

---

Market-Specific Notes

| Market | Key considerations |

|--------|-------------------|

| SE | Swedish customers use specific equestrian terminology — flag any anglicisms |

| DE | Formal tone preferred; compound nouns are critical (get them right) |

| FR | Formal vs. informal (vous/tu) — default to formal for product listings |

| PL | Gendered nouns — flag if translation mismatches product gender |

| NL | Often similar to German but distinct — do not accept DE copy for NL |
