INTERNAL ONLY — DO NOT SHARE WITH CLIENT

**Date**: 2026-03-29

---

WHY THIS STRUCTURE IS DIFFERENT

The previous contract family (v3 through v6) started from this premise:

> *"Preserve MBM's absolute ownership of Company IP."*

That framing made Meridian the party carving out exceptions from a blanket transfer. Every revision — no matter how well-drafted — was negotiating from the wrong starting position.

This MSLA starts from the opposite premise:

> *"Meridian owns everything. Client buys defined access."*

| Old Model | New Model |

|-----------|-----------|

| Client owns all work product by default | Meridian owns all work product by default |

| Meridian proves what it keeps | Client's rights are defined by what it buys |

| IP assignment with carve-outs | IP retention with license grants |

| Exhibit A = "what Contractor keeps" | Exhibit A = "what Meridian owns" (for reference) |

| Price = hourly rate for labor | Price = hourly rate + license tier for rights |

| Buyout is assumed | Buyout is an expensive exception |

| Contractor must justify keeping tools | Client must justify acquiring tools |

The commercial advantage: Meridian now has **four products to sell**, not one:

1. **Services** (hourly rate for labor)

2. **Standard License** (included in base fee)

3. **Extended/Enterprise License** (premium tier)

4. **Buyout** (separate agreement, separate consideration)

This is how software companies, SaaS platforms, and technology contractors structure their agreements when they intend to keep their IP.
