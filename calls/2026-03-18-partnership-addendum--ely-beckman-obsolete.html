Question: Where Does Glassy Fit?

Option A: Under Bitler's Sales Organization

**Structure**: Glassy operates as a Sales Rep within Bitler's channel.

```

Revenue from Glassy-sourced deals:

├─ Meridian → 60%

├─ Bitler (override) → 10%

├─ Community Leader → 15% (Luis/Nate/Andrew if applicable)

└─ Glassy (closer) → 15%

```

**Pros**:

• Clean — no changes to existing term sheet

• Bitler manages Glassy, not Meridian

• Bitler is incentivized to support Glassy's success (his 10% override depends on it)

**Cons**:

• Glassy only gets 15% — may feel too low given his network contribution

• Glassy might function more as a Community Leader (network access) than a Sales Rep (deal closer)

• If Glassy both sources and closes, he gets 15% while Bitler gets 10% for overhead management

Option B: Glassy as Community Leader

**Structure**: Glassy's network (Amazon sellers, CPA contacts, business brokers) qualifies him as a Community Leader who provides the audience and trust.

```

Revenue from Glassy-network deals:

├─ Meridian → 60%

├─ Bitler (override) → 10%

├─ Glassy (audience) → 15%

└─ Separate closer → 15%

```

If Glassy both provides the network AND closes:

```

├─ Meridian → 60%

├─ Bitler (override) → 10%

└─ Glassy (source + close) → 30%

```

**Pros**:

• Recognizes Glassy's primary value (network, not closing)

• If Glassy also closes, he gets 30% — strong incentive

• Cleaner role definition

**Cons**:

• Bitler's 10% override means Glassy's 30% + Bitler's 10% = 40%, which is the full pool

• No room for a separate closer if needed

• May create tension if Bitler feels he should get more for introducing Glassy

Option C: Glassy as Direct Partner (Parallel to Bitler)

**Structure**: Glassy operates independently of Bitler's channel with his own override and terms.

```

Revenue from Glassy-originated deals (NOT through Bitler):

├─ Meridian → 60%

(No Bitler override on deals Glassy sources independently)

Revenue from Bitler-originated deals where Glassy closes:

├─ Meridian → 60%

├─ Bitler (override) → 10%

├─ Community Leader → 15%

└─ Glassy (closer) → 15%

```

**Pros**:

• Glassy is motivated to source deals independently (higher cut)

• Clear attribution: who sourced, who closed

• Scales — Bitler and Glassy can each build their own networks

**Cons**:

• More complex tracking

• Potential overlap / conflict on sourcing attribution

• Bitler might feel bypassed if Glassy activates contacts Bitler introduced him to

• Need clear rules on who "sourced" a deal (first touch? relationship basis?)
