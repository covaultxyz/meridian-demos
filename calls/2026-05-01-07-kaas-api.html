Lex Terminal — KaaS API & Cloudflare Tunnel Runbook

**External URL**: `lex.speedoftrust.ai` (or TBD subdomain)

**Internal service**: FastAPI at `localhost:8443` on big-white

**Access model**: Zero open ports — Cloudflare Tunnel only

This runbook covers Phase 6 of the master checklist. The KaaS FastAPI is Phase 4 work per

the terminal blueprint — Tier 1 API endpoints. Phase 0-3 must be complete first.

---

Phase 1 — Cloudflare Tunnel Setup

1.1 Install cloudflared

```bash

wget -q https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb

dpkg -i cloudflared-linux-amd64.deb

cloudflared --version

```

1.2 Authenticate (operator must complete this in a browser session)

```bash

This opens a browser link. Operator must complete OAuth in browser.

cloudflared tunnel login

Creates /root/.cloudflared/cert.pem on success

```

1.3 Create tunnel

```bash

cloudflared tunnel create lex-kaas

Records the tunnel UUID — note it for step 1.4

Route DNS to tunnel

cloudflared tunnel route dns lex-kaas lex.speedoftrust.ai

```

1.4 Configure tunnel

```bash

mkdir -p /etc/cloudflared

cat > /etc/cloudflared/config.yml << EOF

tunnel: <tunnel-uuid-from-step-1.3>

credentials-file: /root/.cloudflared/<tunnel-uuid>.json

ingress:

• hostname: lex.speedoftrust.ai

service: http://localhost:8443

• service: http_status:404

EOF

```

1.5 Install as system service

```bash

cloudflared service install

systemctl enable cloudflared

systemctl start cloudflared

systemctl status cloudflared

```

Phase 2 — KaaS FastAPI Service

2.1 Health endpoint placeholder (deploy now, full API is Phase 4 work)

The KaaS FastAPI lives at `gateway/caselaw_api.py` (GH#2613). For initial deploy,

a health endpoint is sufficient to validate the tunnel.

```bash

Check if the API exists

ls /opt/meridian/current/gateway/caselaw_api.py

If exists → configure systemd unit (Phase 2.2)

If not → deploy placeholder (Phase 2.3)

```

2.2 Systemd unit for the API service

```bash

cat > /etc/systemd/system/lex-kaas-api.service << 'EOF'

[Unit]

Description=Lex KaaS FastAPI

After=network.target neo4j.service

Wants=neo4j.service

[Service]

Type=simple

EnvironmentFile=/etc/lex/env

WorkingDirectory=/opt/meridian/current

ExecStart=/opt/lex/.venv/bin/uvicorn gateway.caselaw_api:app --host 127.0.0.1 --port 8443

Restart=on-failure

RestartSec=10

User=meridian

StandardOutput=journal

StandardError=journal

SyslogIdentifier=lex-kaas-api

[Install]

WantedBy=multi-user.target

EOF

systemctl daemon-reload

systemctl enable lex-kaas-api

systemctl start lex-kaas-api

```

2.3 Placeholder if caselaw_api.py not yet built

```bash

cat > /opt/lex/placeholder_api.py << 'EOF'

from fastapi import FastAPI

import os

app = FastAPI(title="Lex KaaS API", version="0.1.0")

@app.get("/health")

async def health():

return {

"status": "ok",

"terminal": "lex",

"workspace_id": os.environ.get("WORKSPACE_ID", "lex"),

"api_version": "0.1.0-placeholder"

}

@app.get("/")

async def root():

return {"message": "Lex Legal Intelligence API — Phase 4 endpoints coming"}

EOF

```

Update systemd unit to use placeholder:

```bash

sed -i 's|gateway.caselaw_api:app|lex.placeholder_api:app|' \

/etc/systemd/system/lex-kaas-api.service

systemctl daemon-reload

systemctl restart lex-kaas-api

```

```bash
Expected: same JSON

```

Phase 4 — Tier 1 API Endpoints (Phase 4 work from blueprint)

Once `gateway/caselaw_api.py` is built, these endpoints should be live:

| Method | Path | Description |

|--------|------|-------------|

| GET | `/health` | Service health |

| POST | `/search` | BM25 case search with jurisdiction filter |

| GET | `/case/{case_id}` | Full case record by ID |

| GET | `/citations/{case_id}` | Forward/backward citation graph |

| GET | `/judge/{name}` | Judge ruling profile |

| GET | `/jurisdictions` | Available court list |

Authentication for KaaS customers: API key header (`X-Lex-API-Key`) — managed by Sterling.

Common Failures

Tunnel auth expired

```bash

Check tunnel status

cloudflared tunnel info lex-kaas

If "not found" or "error": re-authenticate

cloudflared tunnel login # browser auth required

systemctl restart cloudflared

```

Port 8443 not listening

```bash

ss -tlnp | grep 8443

If empty: the API service isn't running

systemctl status lex-kaas-api

journalctl -u lex-kaas-api -n 50

```

DNS not routing to tunnel

```bash

Check CF Tunnel DNS route

cloudflared tunnel route list lex-kaas

Should show lex.speedoftrust.ai → lex-kaas

If missing, re-add:

cloudflared tunnel route dns lex-kaas lex.speedoftrust.ai

```
