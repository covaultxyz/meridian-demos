Key Agreements

1. Sales Strategy — "Building at the Speed of Trust"

• **Not cold outreach**. All sales through trusted intermediaries (CPAs, attorneys, business brokers)

• Hub-and-spoke model: win over one hub (CPA with 500 clients), the spoke referrals follow

• Low-hanging fruit first (voice AI, review removal, SOP documentation), then expand into full terminal

2. Three-Tier Sales Motion

| Tier | Product | Entry Point | Revenue |

|------|---------|-------------|---------|

| 1 | Quick wins | Voice AI receptionist, Amazon review removal, SOP documentation | $175-$2,000/engagement |

| 2 | Terminal deployment | Full AI operating system (research, dashboards, daemon shells, Telegram interface) | $2,000-$5,000/mo |

| 3 | Knowledge-as-a-Service | API access to specialized MIG brains (compliance, sentiment, supply chain) | Custom pricing |

3. Revenue Model

• Consensus leaning toward **performance-based** to reduce buyer friction

• "Easier to make money then help them pay" — show value before extracting payment

• Hybrid acceptable: small retainer + performance kicker

• For pre-acquisition play: take % of increased sale multiple (could be $200K+ on a $50K-$100K engagement)

4. Deployment Model

• **Cloud VPS** (default): Managed by Meridian, clients access via Telegram + dashboard

• **On-prem Mac Mini clusters**: 5 Mac Minis for the price of 1 Mac Studio, mesh-native

• **Hybrid**: Local compute + cloud backup for larger firms

• Key: clients never touch the code. Telegram + dashboard are the interfaces.

5. IP Protection (Bitler raised this — sharp thinking)

• **Moat 1**: MIG brain (7,000+ nodes, massive ontology, years of curation) — can't be downloaded via API

• **Moat 2**: Managed VPS — clients access tools, not code

• **Moat 3**: Architecture knowledge — even with code, building this from scratch takes years

• **Moat 4**: Trust network — relationship bank can't be cloned

• **Risk acknowledged**: A technical buyer could reverse-engineer from a VPS. Mitigation: keep clients on Telegram + dashboard interfaces, serve MIG via API only.
