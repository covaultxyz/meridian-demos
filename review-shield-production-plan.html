14 Build Plans

Production Readiness Plans

Each plan is a standalone build spec with schema, code patterns, acceptance criteria, and files to create.

01

Neo4j Schema & Migration

2–3 days

Define graph schema for the review removal business. 5 node types (ReviewClient, ReviewProduct, ProductReview, ReviewAppeal, ReviewInvoice) with 6 relationship types. Parse 50+ Excel files into Neo4j.

Nodes: ~10 clients, ~60 products, ~500 reviews, ~135 appeals

Migration: brain/migrations/055_review_removal_schema.py

Import: scripts/import_review_data.py (openpyxl → Cypher)

CURSOR AGENT OFFLOADABLE

02

Keyword Database

1 day

Structure 200+ trigger keywords from the existing ChatGPT bot into machine-readable YAML. 15 violation categories with regex patterns for masked profanity.

hostilefalse_claimsprofanityshippingpricingfakecompetitorhealth_claims+7 more

Pre-filter gate: Reviews with 0 keyword matches skip LLM classification (saves ~60% of tokens)

03

JetStream Streams

0.5 day

5 JetStream streams with durable consumers, message schemas, and dead letter routing. Workqueue retention for processing stages, limits retention for outcome history.

Ack wait: 120s (Claude classification may be slow)

Max deliver: 3 retries before DLQ

Outcome retention: 365 days (training data)

NO DEPENDENCIES

04

Review Scanner

3–4 days

Automated detection of new 1–3 star reviews on monitored ASINs. Two paths: Helium 10 integration (preferred) and direct Amazon scraping (fallback).

Cadence: Daily pulse sentinel

Dedup: By Amazon review_id

Rate limit: 1 request/2s to Amazon

Files: integrations/helium10.py, integrations/amazon_reviews.py

DISCOVERY NEEDED: Helium 10 API access?

05

Violation Classifier

3–4 days

Claude-powered review analysis replacing ChatGPT keyword matching. Two-stage: cheap keyword pre-filter, then full LLM classification with few-shot examples from historical successes.

Model: Haiku (bulk) / Sonnet (borderline)

Cost: ~$0.50/day for 100 reviews (Haiku)

Gate: strength ≥ moderate AND confidence ≥ 0.6

Target: 80%+ recall, 70%+ precision

KEY DIFFERENTIATOR: Context vs keywords

06

Appeal Generator

2–3 days

Letter generation using Claude with strict format enforcement, hallucination guard (quotes must exist in review), and quality gate. 10 format rules, zero tolerance for deviation.

Model: Sonnet (prose quality matters)

Quality gate: Format + hallucination + length + ASIN check

Retry: 2x with error feedback on gate failure

Few-shot: Pull from MIG by violation category

TRAINED ON 135 HISTORICAL SUCCESSES

07

Submission Pipeline

3–5 days

Three tiers of implementation. Start with human-in-the-loop (Telegram approval), graduate to email, then full Seller Central automation.

1

Telegram approval — operator reviews + manually submits

2

Email submission — auto-send to community-abuse@amazon.com

3

Seller Central automation — Playwright browser scripting

08

Outcome Tracker

1–2 days

Daily pulse sentinel that checks if appealed reviews have been removed from Amazon. URL existence check with rate limiting. Triggers invoice on successful removal.

Detection: HTTP 404 or "no longer available" text

Expiry: >180 days pending → mark as rejected

Rate limit: 2s between Amazon URL checks

Analytics: Removal rate by category, avg days, trends

09

Invoicing System

2–3 days

Auto-generate PDF invoices on successful removal. Sequential numbering (RR-YYYY-NNNN), email delivery, payment tracking via Telegram commands.

Pricing: $150 standard / $175–$225 premium + custom overrides

PDF: ReportLab generation with branded template

Commands: /paid, /overdue, /revenue via Telegram

Future: Stripe webhook for auto-payment detection

10

Client Onboarding

2–3 days

Telegram bot flow: /register → collect info → Seller Central access instructions → /addproduct <ASIN> → monitoring begins. Under 10 minutes from first contact to live monitoring.

Operator commands: /clients, /confirm_access, /deactivate, /reactivate
