The Three Importance-10 Memories

These form a complete stack. Each answers a different question.

1. Conversation Architecture (the FLOW)

Search: mig_search("welch diagnose cleveland redirect unicorn discover")

WHAT QUESTION DOES IT ANSWER?
"In what order do I run the conversation?"

THE SEQUENCE:
Step 1: WELCH DIAGNOSE
  → Identify what emotional chain the prospect is already running
  → Use subset of 16-step coaching sequence
  → Key question: "When was the first time you felt [this problem]?"

Step 2: CLEVELAND REDIRECT  
  → Chunk UP to the prospect's abstract criteria (security, freedom)
  → Anchor the emotion attached to that criteria
  → Chunk DOWN: redefine what that criteria means → your product

Step 3: UNICORN DISCOVER
  → Lead the prospect to realize the connection themselves
  → Ask questions that connect dots — don't tell
  → Use isomorphic metaphor: third-person story that mirrors their situation

2. Criteria Elicitation (the ENGINE)

Search: mig_search("criteria elicitation construction manual")

WHAT QUESTION DOES IT ANSWER?
"How do I find out what the prospect actually wants?"

THE PATTERN:
  "What's important about [context]?" → Level 1 (what they think they want)
  "What's important about [their answer]?" → Level 2 (one meta level up)
  "What's important about [that]?" → Level 3 (values — controls everything below)

RULES:
  - Each level UP controls everything below it
  - Listen for towards/away polarity ("I don't want to be broke" = away-from)
  - Soften with: "I'm curious..." / "Tell me more" / "Why now?"
  - NEVER tell them their criteria. Get THEM to tell you.
  - Listen for EMPHASIS — start questioning from the emphasized item

3. Language Generation Pipeline (the OUTPUT)

Search: mig_search("agent vox language generation pipeline")

WHAT QUESTION DOES IT ANSWER?
"How does the AI construct its responses?"

TWO-PASS ARCHITECTURE (Write-Then-Layer):
  PASS 1 — SURFACE: Generate conversational content
    - The story, the pitch, the response
    - Must be genuine, emotionally invested, structurally sound
    - Follows nested story structure when multi-turn

  PASS 2 — DEEP STRUCTURE: Layer in influence architecture
    - Embedded commands: 2-4 words, present tense, targeting elicited criteria
    - Analogical marking: tone shifts in voice synthesis (pause + deeper + volume)
    - Sensory stacking: visual + auditory + kinesthetic language
    - Command sequence: emotional commands first, action commands later
    - Pattern density: sustained commands across conversation (not one-off)

FOR WRITTEN CHANNELS (email, chat):
  Mark commands with subtle visual differentiation (bold, ellipsis, slight font change)

AI Sales Coach / Trainer

Ready for Phase 4. The Welch 16-step sequence IS the coaching state machine.

DIAGNOSTIC PHASE (Steps 1-9):
  1. "When was the first time you felt [X]?"
  2. "How'd that make you feel?"
  3. "Take me deeper on the feeling"
  4. "Did you have to work harder?"
  5. "Do you struggle with that today?"
  6. "What choices does it cause you to make?"
  7. "Anything else?"
  8. "Project these choices — what outcomes?"
  9. Mirror the full causal chain back

REPROGRAMMING PHASE (Steps 10-16):
  10. "Why did they pay you $[amount]?"
  11. "Turn the story into a principle"
  12. "Can you get them the result? How?"
  13. "You can bail yourself out — true or false?"
  14. "Where are you right now physically?"
  15. "How much did you make then vs now?"
  16. "Progress or table stakes?"

Evaluation: RAG 5-point rubric (citation, contradiction, incorrect info, conversational quality, info leakage)

Extraction Quality Standards

  1. wc -l before reading — know the file size
  2. 50%+ read minimum (100% for named frameworks)
  3. mig_search() before mig_remember() — dedup
  4. Depth pressure on every extraction — cross-domain synthesis only comes from depth
  5. Noise Gate: "Am I witnessing or generating?"
  6. If you can't write a full audit artifact, the extraction is too thin
